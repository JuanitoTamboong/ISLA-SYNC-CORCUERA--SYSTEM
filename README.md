# ISLA-Sync Corcuera (Android WebView)

A mobile-first, single-page web application wrapped in **Capacitor** for **Corcuera community management**. The app is designed around two roles:

- **Residents** – submit reports, browse news/tourist information, manage profile.
- **Admin** – review/resolve reports, publish news, and manage tourist spot content.

This repository contains the web UI (HTML/CSS/JS) plus the Capacitor/Android project.

---

## ✨ Highlights

- **WebView + Capacitor**: ship one codebase as a native Android app.
- **Supabase Auth**: sign up/login and secure reset password flows.
- **Realtime (Supabase)**: updates for content such as **reports** and **news**.
- **Google Maps**: view and interact with tourist places/report pinning.
- **Material-style UI**: consistent buttons, cards, and touch-friendly components.

---

## 📱 System Overview (How it works)

1. **User opens the app** (Android WebView).
2. The web app loads the correct page (e.g. `pages/login.html`, `pages/news.html`, `pages/report.html`).
3. Client-side JavaScript handles:
   - Authentication (Supabase)
   - Role-based navigation (Admin vs Resident)
   - Data reads/writes to Supabase
   - Realtime updates (where enabled)
4. Admin actions update the Supabase database, and connected clients update automatically.

## 🚀 Quick Start

### 1) Install dependencies

```bash
npm install
```

### 2) Configure Capacitor

Capacitor config is in:

- `capacitor.config.json`

This project uses a **WebView** configuration that allows mixed content:

- `android.allowMixedContent = true`
- `server.androidScheme = https`

### 3) Configure Supabase

This codebase currently includes Supabase URL/key inside JS files (example shown in password reset scripts).

To run locally, set your **Supabase** credentials in the relevant scripts (or refactor into a shared config module if you want).

Common tasks in Supabase:
- Create tables required by the app (reports, news, tourist spots, etc.)
- Ensure **Auth** is enabled
- Enable **Realtime** for tables that should push live updates

> The exact schema is not included as a single file in this repo listing you shared, so you should verify which migrations/SQL are present (often in `backup-sql/` or a `schema.sql`).

### 4) Configure Google Maps

The app loads Google Maps from the browser context. Provide your API key in the file that injects the Maps script (commonly `index.html` or map-related code).

## 🔐 Authentication & Password Reset (Supabase)

The project implements reset password flows for both users and admin.

Examples:

- `pages/reset-password.html` with logic in `js/reset-password.js`
- `pages/admin/admin-reset-password.html` with logic in `js/admin/admin-reset-password.js`

These pages:
- Load Supabase SDK
- Validate new password requirements
- Confirm password match
- Call `supabaseClient.auth.updateUser({ password })`
- Handle expired/invalid reset sessions and redirect to the admin/user login page

## 🧩 Data & Realtime (Supabase)

The app expects certain Supabase tables to exist and (optionally) be configured for realtime updates. Typical realtime behavior:
- Admin resolves a report → resident sees status change
- New news item published → list updates without refresh

If realtime does not work:
- Confirm Realtime is enabled in Supabase for the target tables
- Verify channel names / subscription code in JS

## 🛠 Troubleshooting

### 1) Supabase not connecting
- Confirm URL and anon/public key are correct
- Ensure internet access is allowed from the device

### 2) WebView mixed-content issues
- This repo already enables `allowMixedContent` in Capacitor config
- If you disable it later, serve everything over HTTPS

### 3) Maps not loading
- Verify Google Maps API key
- Ensure Maps JS API is enabled in Google Cloud

### 4) Reset password link issues
- Reset links expire → redirect user to login and request a new reset email
- Make sure the Supabase email templates are configured correctly
